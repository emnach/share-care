# Contributing to Share & Care

First off, thank you for considering contributing to Share & Care! It's people like you that make Share & Care such a great tool for fighting food waste.

## Code of Conduct

This project and everyone participating in it is governed by respect, kindness, and collaboration. By participating, you are expected to uphold these values.

## How Can I Contribute?

### Reporting Bugs

Before creating bug reports, please check the issue list as you might find out that you don't need to create one. When you are creating a bug report, please include as many details as possible:

* **Use a clear and descriptive title**
* **Describe the exact steps which reproduce the problem**
* **Provide specific examples to demonstrate the steps**
* **Describe the behavior you observed and what behavior you expected**
* **Include screenshots if possible**
* **Include your browser name and version**

### Suggesting Enhancements

Enhancement suggestions are tracked as GitHub issues. When creating an enhancement suggestion, please include:

* **Use a clear and descriptive title**
* **Provide a step-by-step description of the suggested enhancement**
* **Provide specific examples to demonstrate the steps**
* **Describe the current behavior and explain which behavior you expected to see instead**
* **Explain why this enhancement would be useful**

### Pull Requests

* Fill in the required template
* Follow the existing code style (see Style Guide below)
* Include screenshots and animated GIFs in your pull request whenever possible
* End all files with a newline
* Test your changes thoroughly

## Style Guide

### HTML
* Use semantic HTML5 elements
* Indent with 2 spaces
* Use double quotes for attributes
* Close all tags properly

### CSS
* Follow the existing naming conventions
* Keep selectors simple and specific
* Group related properties
* Use comments for complex sections
* Maintain the mobile-first approach

### File Organization
* Keep HTML files self-contained with inline CSS
* Use meaningful file names (e.g., `01-role-selection.html`)
* Maintain the existing directory structure

### Design Guidelines
* Follow the established design system (see docs/DESIGN_SYSTEM.md)
* Maintain the 390x844px mobile viewport
* Use the defined color palette
* Follow the 8pt spacing grid
* Keep accessibility in mind (contrast ratios, font sizes)

## Development Process

1. **Fork the repository** and create your branch from `main`
2. **Make your changes** following the style guide
3. **Test your changes** on multiple browsers/devices
4. **Commit your changes** with clear, descriptive messages
5. **Push to your fork** and submit a pull request

### Commit Messages

* Use the present tense ("Add feature" not "Added feature")
* Use the imperative mood ("Move cursor to..." not "Moves cursor to...")
* Limit the first line to 72 characters or less
* Reference issues and pull requests liberally after the first line

Examples:
```
Add donor profile statistics section

- Display meals donated count
- Show CO2 savings metric
- Add visual badges for achievements

Fixes #123
```

## Testing

Before submitting a pull request, please test your changes:

* [ ] Test on Chrome (latest)
* [ ] Test on Firefox (latest)
* [ ] Test on Safari (if on Mac)
* [ ] Test on mobile devices (or use browser dev tools)
* [ ] Check all interactive elements work
* [ ] Verify no console errors
* [ ] Test with different screen sizes

## Project Goals

Keep these goals in mind when contributing:

1. **Simplicity** - Keep the code simple and maintainable
2. **Accessibility** - Make it usable for everyone
3. **Performance** - Keep it fast and lightweight
4. **Mobile-First** - Optimize for mobile experience
5. **No Dependencies** - Maintain zero external dependencies
6. **Self-Contained** - Each HTML file should work standalone

## Need Help?

Don't hesitate to ask for help! You can:
* Open an issue with the "question" label
* Reach out in the discussions section
* Comment on the issue you're working on

## Recognition

Contributors will be recognized in the README.md file. Thank you for helping make Share & Care better!
